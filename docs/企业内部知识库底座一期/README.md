# 企业内部知识库底座一期

本目录沉淀 WeKnora 作为“企业内部知识库底座”的一期项目资料。资料维护原则是：只保留当前决策、当前范围、当前待办；已完成的阶段性任务不再放在执行清单里，避免误导后续开发。

## 商业目标

WeKnora 一期的商业目的不是做一个泛 AI 平台，而是帮助企业把分散文档、FAQ 和业务知识沉淀成可检索、可问答、可持续维护的内部知识底座。

核心价值：

- 降低员工找资料、问同事、重复答疑的时间成本。
- 让业务人员可以自己维护知识库和 FAQ，不依赖研发反复处理内容。
- 通过引用来源、检索、问答和共享能力，提高知识答案的可信度和复用率。
- 为后续 Agent、外部数据源、IM、GraphRAG 等能力保留扩展空间，但不让它们干扰一期主流程。

## 当前角色模型

一期只保留两个角色：

- 平台超级管理员：负责平台级配置、租户管理、系统运行和全局能力边界。
- 普通租户：租户自己的管理员和使用者，不再拆分“租户管理员”这个独立角色。

## 维护规则

- 已完成的任务不继续保留在执行清单中。
- 新需求必须先说明业务价值、用户场景和验收标准，再进入开发。
- 优先优化用户主链路：创建知识库、导入内容、检索、问答、FAQ、共享、基础配置。
- 技术优化服务于用户体验，不单独以“堆功能”作为目标。
