## 一期目标

- 建立企业统一知识入口
- 支持文档与 FAQ 的沉淀、检索和问答
- 支持组织内共享与基础角色权限
- 支持私有化部署与基础安全运维

## 一期边界

- 不以研发知识中台为目标
- 不纳入 GitHub、GitLab、Jira、Confluence 等深度集成
- 不纳入源系统 ACL 继承
- 不纳入 MCP、GraphRAG、复杂 Agent、全渠道 IM 的生产开放

## 一期开发原则

- 先做能力收敛，再做体验优化
- 先保证主流程稳定，再扩展高级能力
- 前台坚持“知识库产品”表达，不暴露过多技术概念
