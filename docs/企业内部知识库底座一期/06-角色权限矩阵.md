# 角色权限矩阵

## 角色定义

一期只保留两个角色。

| 角色 | 定义 | 核心目标 |
|---|---|---|
| 平台超级管理员 | 平台运营和全局配置负责人 | 管租户、管平台配置、控风险、看运行状态 |
| 普通租户 | 租户自己的管理员和使用者；“租户管理员”只是称呼，不是独立角色 | 建知识库、维护 FAQ、搜索问答、管理本租户知识 |

## 权限矩阵

| 功能 | 平台超级管理员 | 普通租户 |
|---|---|---|
| 登录和进入平台 | 是 | 是 |
| 查看平台总览 | 是 | 否 |
| 管理租户 | 是 | 否 |
| 查看全部租户数据 | 是，限平台运营场景 | 否 |
| 管理本租户知识库 | 是 | 是 |
| 上传和维护本租户文档 | 是 | 是 |
| 维护本租户 FAQ | 是 | 是 |
| 搜索本租户知识 | 是 | 是 |
| 发起智能问答 | 是 | 是 |
| 查看引用来源 | 是 | 是 |
| 管理本租户共享空间 | 是 | 是 |
| 平台模型、存储、检索配置 | 是 | 否 |
| 系统健康状态和运行配置 | 是 | 否 |
| MCP、GraphRAG、高级工具配置 | 谨慎开放 | 否 |

## 产品原则

- 普通租户不再拆分编辑者、空间管理员、租户管理员等子角色。
- 普通租户界面只突出“建、传、搜、问、改、共享”的业务动作。
- 平台超级管理员界面只突出“租户、配置、运行、安全”的平台动作。
- 一期不在前台暴露复杂模型参数、工具链编排和实验性能力。
