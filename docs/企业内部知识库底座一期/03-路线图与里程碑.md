# 路线图与里程碑

## 当前阶段判断

项目已经从“能力梳理和功能改造”进入“一期试点前稳定化”阶段。后续优先级不再按技术模块堆叠，而是按业务闭环排序：

1. 普通租户能否独立完成知识库建设和问答使用。
2. 平台超级管理员能否控制租户、配置和系统风险。
3. 试点用户能否看懂结果、信任答案、发现问题并反馈。
4. 运维人员能否定位导入、解析、检索和问答链路问题。

## M1：试点前稳定化

目标：让核心业务链路稳定、清晰、可验收。

交付物：

- 普通租户首页、知识库、FAQ、搜索、问答、设置入口清晰。
- 平台超级管理员入口与普通租户入口隔离。
- 已完成任务清单从项目资料中清理，只保留当前有效待办。
- 前端首屏和大资源加载保持在可接受范围。
- 登录、导入、解析、检索、问答、引用来源、共享流程通过冒烟验证。

验收标准：

- 试点用户不需要理解底层模型、向量库、MCP、GraphRAG 等概念，也能完成主要操作。
- 平台超级管理员和普通租户看到的菜单、首页和设置项符合角色边界。

## M2：试点运营

目标：用真实业务知识验证产品价值。

交付物：

- 导入首批高价值文档和 FAQ。
- 建立问题样本集：高频问题、无结果问题、低质量答案、权限边界问题。
- 形成每周运营复盘：命中率、无结果率、用户反馈、待补充知识。
- 优化空状态、错误提示、导入进度和问答引用体验。

验收标准：

- 普通租户可以根据反馈持续补充知识。
- 高频问题可以被有效检索和回答。
- 低质量答案有可追踪的原因和改进路径。

## M3：一期上线

目标：形成可交付、可运维、可扩展的一期版本。

交付物：

- 上线说明和用户使用指引。
- 生产配置、备份恢复、关键服务健康检查和告警策略。
- 一期范围内的功能验收记录。
- 二期候选需求池，不混入一期执行清单。

验收标准：

- 业务方确认一期核心价值闭环成立。
- 平台进入稳定运营状态。

## 二期候选方向

以下能力保留为二期或后续增强，不作为一期上线阻塞项：

- 更强的数据源同步和连接器。
- Agent 编排和复杂工具链。
- GraphRAG 生产级图谱问答。
- MCP 平台化开放。
- 多渠道 IM 全量接入。
- 运营分析看板和答案质量评估体系。
