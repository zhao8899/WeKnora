# 一期建设方案

## 建设目标

- 建立企业内部统一知识入口
- 支持文档型知识库和 FAQ 型知识库
- 支持 Web 端搜索与智能问答
- 支持组织内共享和基础权限控制
- 支持私有化部署、备份、监控和基础安全

## 一期定位

一期定位为“企业内部知识库底座”，不是企业级研发知识中台，也不是通用 AI 平台。

## 一期范围

- OIDC 登录与基础用户体系
- 组织、共享空间、成员邀请与审批
- 文档上传、文件夹导入、URL 导入、在线录入
- 文档解析、索引、检索、引用来源展示
- FAQ 创建、编辑、维护
- 多轮问答与建议问题
- 管理员级模型、检索、存储、解析配置
- 私有化部署、日志、备份、健康检查

## 一期不纳入范围

- 研发知识中台能力
- 深度外部系统集成
- 源系统细粒度权限继承
- MCP 大规模接入
- Web Search 默认开放
- GraphRAG 生产启用
- 复杂 Agent 编排

## 一期成功标准

- 普通员工能在 3 步内完成搜索或提问
- 编辑者能独立完成知识录入和 FAQ 维护
- 共享空间能支撑部门级知识协作
- 主流程稳定，权限边界清晰
- 试点部门认可可用性并愿意持续使用
