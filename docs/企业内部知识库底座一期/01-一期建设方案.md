# 一期建设方案

## 建设目标

- 建立企业内部统一知识入口。
- 支持文档型知识库和 FAQ 型知识库。
- 支持 Web 端搜索、智能问答和引用来源查看。
- 支持普通租户在本租户内维护知识、FAQ 和共享空间。
- 支持平台超级管理员进行租户、平台配置、系统运行和安全边界管理。
- 支持私有化部署、日志、备份、健康检查和基础安全运维。

## 一期定位

一期定位为“企业内部知识库底座”，不是企业级研发知识中台，也不是通用 AI 平台。

产品表达应围绕业务用户能理解的动作展开：建知识库、传资料、维护 FAQ、搜索、提问、看来源、共享。

## 一期范围

- 登录、注册或 OIDC 登录。
- 两角色模型：平台超级管理员、普通租户。
- 知识库创建、文档上传、文件夹导入、URL 导入、在线录入。
- 文档解析、索引、搜索、引用来源展示。
- FAQ 创建、编辑、批量导入、校验和维护。
- 多轮问答、建议问题、答案来源和证据表达。
- 本租户共享空间和知识协作。
- 平台级模型、检索、存储、解析配置。
- 私有化部署、日志、备份、健康检查。

## 一期不纳入范围

- 复杂研发知识中台能力。
- 深度外部系统集成和源系统 ACL 继承。
- MCP 大规模平台化开放。
- Web Search 面向普通租户默认开放。
- GraphRAG 生产级开放。
- 复杂 Agent 编排和工具链配置。
- 多渠道 IM 全量接入。

## 一期成功标准

- 普通租户可以独立完成知识库建设、内容导入、FAQ 维护、搜索和问答。
- 普通租户能理解系统下一步建议，遇到失败时知道如何处理。
- 平台超级管理员可以管理租户、配置平台能力并查看关键运行状态。
- 问答结果具备可追溯来源，无法回答时有明确反馈。
- 主流程稳定，权限边界清晰。
- 试点业务方认可可用性，并愿意持续把真实业务知识沉淀进系统。
