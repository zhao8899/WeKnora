# 当前待办与上线检查

本文只维护“尚需处理”的事项。已完成的开发、优化和历史任务不再保留在本清单中。

## P0：试点前必须处理

- 核对平台超级管理员与普通租户的菜单、路由、首页、设置页是否完全符合两角色模型。
- 完成普通租户主链路冒烟：创建知识库、导入文档、维护 FAQ、搜索、问答、查看引用、共享。
- 完成平台超级管理员主链路冒烟：租户管理、平台配置、系统运行状态、关键风险操作。
- 梳理空状态和错误提示：导入失败、解析失败、无搜索结果、无引用来源、模型未配置。
- 清理产品资料中过期角色、已完成任务和与一期无关的功能承诺。
- 确认生产环境基础项：配置、日志、备份、健康检查、告警、回滚方案。

## P1：试点期建议处理

- 优化新租户首次使用引导，减少用户不知道先做什么的问题。
- 优化答案可信度表达，包括引用来源、检索片段、无法回答时的提示。
- 优化 FAQ 批量导入和校验反馈，让业务人员能自行修复数据问题。
- 建立试点问题样本集，持续记录无结果、低质量答案和知识缺口。
- 汇总租户常用操作入口，把低频配置从主路径弱化。

## P2：暂不纳入一期上线阻塞

- GraphRAG 生产级能力开放。
- MCP 平台化开放。
- 多 Agent 编排和复杂工具链配置。
- 多外部系统深度同步。
- 多渠道 IM 全量接入。
- 高级运营分析和自动化质量评估。

## 上线检查

- 登录、注册或 OIDC 流程可用，异常提示明确。
- 普通租户能独立完成知识导入、检索、问答和 FAQ 维护。
- 平台超级管理员能完成平台级配置和租户管理。
- 权限边界清晰，无明显越权路径。
- 常见文档类型解析成功，失败时可追踪原因。
- 问答结果能展示来源或明确说明无法回答。
- 关键异步任务有进度、终态和失败提示。
- 备份恢复、日志、健康检查和告警已验证。
- 试点业务方完成验收并确认上线范围。
