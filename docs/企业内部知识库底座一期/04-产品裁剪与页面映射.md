# 产品裁剪与页面映射

## 裁剪原则

一期产品不追求展示全部技术能力，而是让两个角色能快速完成自己的核心任务。

- 普通租户优先看到业务动作：建知识库、导入内容、维护 FAQ、搜索、问答、共享。
- 平台超级管理员优先看到平台动作：租户、配置、运行状态、安全边界。
- 低频、实验性、研发导向能力不进入普通租户主路径。
- 已实现但不适合一期主流程的能力可以保留在代码层或后台配置中，不作为前台卖点。

## 普通租户主路径

| 页面或入口 | 是否保留 | 用户价值 | 优化方向 |
|---|---|---|---|
| 首页 | 保留 | 告诉用户下一步该做什么 | 聚合常用入口、最近知识库、待处理导入、快速提问 |
| 知识库 | 保留 | 管理企业知识资产 | 强化创建、导入、解析状态、内容维护 |
| FAQ | 保留 | 沉淀高频标准问答 | 强化批量导入、校验反馈、重复问题处理 |
| 知识搜索 | 保留 | 快速查资料 | 强化无结果反馈、筛选、引用来源 |
| 智能问答 | 保留 | 用自然语言消费知识 | 强化答案可信度、来源、无法回答提示 |
| 共享空间 | 保留 | 管理本租户知识协作 | 保持简单，不拆复杂子角色 |
| 通用设置 | 保留 | 语言、主题、基础偏好 | 只保留用户能理解的配置 |

## 平台超级管理员主路径

| 页面或入口 | 是否保留 | 用户价值 | 优化方向 |
|---|---|---|---|
| 平台总览 | 保留 | 查看平台运行和租户概况 | 聚合租户、用量、风险和状态 |
| 租户管理 | 保留 | 管理租户生命周期 | 强化创建、停用、配额、状态 |
| 系统设置 | 保留 | 控制平台级能力 | 聚合模型、存储、检索、认证等配置 |
| 使用审计 | 保留 | 支撑运营和风险排查 | 优先展示异常、高频、趋势 |

## 一期弱化或隐藏能力

以下能力不删除，但不作为普通租户的一期主路径：

- Agent 独立配置入口。
- MCP 服务配置。
- Web Search 全局调试入口。
- GraphRAG 和知识图谱高级配置。
- 高级模型参数和推理参数。
- 多外部系统深度同步配置。
- 多渠道 IM 接入配置。

## 当前页面映射

| 路由或模块 | 面向角色 | 当前定位 |
|---|---|---|
| `/platform/home` | 普通租户 | 租户工作台 |
| `/platform/dashboard` | 平台超级管理员 | 平台总览 |
| `/platform/knowledge-bases` | 普通租户、平台超级管理员 | 知识库主入口 |
| `/platform/knowledge-search` | 普通租户、平台超级管理员 | 知识搜索入口 |
| `/platform/create-chat` 和 `/platform/chat/:chatid` | 普通租户、平台超级管理员 | 智能问答 |
| `/platform/organizations` | 普通租户、平台超级管理员 | 共享空间或租户协作管理 |
| `/platform/settings` | 按角色展示 | 设置中心，需继续收敛低频配置 |
| `/platform/agents` | 谨慎开放 | 非一期普通租户主路径 |

## 后续体验优化重点

- 首页要回答“我现在该做什么”，而不是展示功能堆砌。
- 导入和解析流程要明确进度、失败原因和下一步处理建议。
- 问答结果要优先解决“我能不能信这个答案”的问题。
- 设置页要减少技术概念，让普通租户只看到必要配置。
- 平台超级管理员与普通租户的路径必须持续隔离，避免角色认知混乱。
