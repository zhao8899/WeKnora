# 项目资料

## 项目定位

WeKnora 当前定位为企业内部知识库底座：把企业内部文档、FAQ、业务资料和问答流程沉淀为可检索、可追溯、可运营的知识服务。

当前阶段优先保证企业内部知识库的一期闭环，而不是扩展成泛 AI 平台。

## 核心价值

- 降低员工找资料、问同事和重复答疑的时间成本。
- 让业务人员可以维护知识库和 FAQ，减少对研发的依赖。
- 通过引用来源、检索、问答和共享能力，提高知识答案的可信度和复用率。
- 为后续 Agent、外部数据源、IM、GraphRAG 和 MCP 能力保留扩展空间，但不让这些能力干扰一期主流程。

## 当前范围

一期主路径只聚焦以下能力：

- 租户和平台基础管理。
- 知识库创建、配置和内容维护。
- 文档导入、解析、预览和重新解析。
- FAQ 维护、批量导入、检索和问答。
- 知识搜索、智能问答、引用来源和无法回答提示。
- 共享空间和基础协作。
- 模型、存储、检索、解析引擎等平台级配置。
- 登录、API Key、OIDC、健康检查和基础安全配置。

以下能力保留为高级能力或后续增强，不作为普通租户的一期主流程：

- GraphRAG 生产级开放。
- MCP 平台化开放。
- 多 Agent 编排和复杂工具链配置。
- 多外部系统深度同步。
- 多渠道 IM 全量接入。
- 高级运营分析和自动化质量评估。
- 面向普通租户的高级模型参数调试。

## 角色边界

一期只保留两个角色。

| 角色 | 定义 | 核心目标 |
|---|---|---|
| 平台超级管理员 | 平台运营和全局配置负责人 | 管租户、管平台配置、控风险、看运行状态 |
| 普通租户 | 租户自己的管理员和使用者；“租户管理员”只是称呼，不是独立角色 | 建知识库、维护 FAQ、搜索问答、管理本租户知识 |

产品原则：

- 普通租户不再拆分编辑者、空间管理员、租户管理员等子角色。
- 普通租户界面只突出“建、传、搜、问、改、共享”的业务动作。
- 平台超级管理员界面只突出“租户、配置、运行、安全”的平台动作。
- 一期不在前台大面积暴露复杂模型参数、工具链编排和实验性能力。

## 商业化状态

当前项目适合企业内网 PoC、试点和私有化交付验证。直接作为多租户公网 SaaS 上线前，还需要完成生产配置、安全边界、监控告警、备份恢复、容量规划和验收流程。

推荐商业包装：

- 企业知识库私有化部署。
- RAG 问答和引用来源。
- 企业内部资料沉淀和 FAQ 维护。
- 租户隔离和平台级配置。
- IM、MCP、GraphRAG 等作为可选增强能力。

## 上线检查

试点前必须确认：

- 平台超级管理员与普通租户的菜单、路由、首页、设置页符合两角色模型。
- 普通租户可以完成创建知识库、导入文档、维护 FAQ、搜索、问答、查看引用和共享。
- 平台超级管理员可以完成租户管理、平台配置、系统运行状态查看和关键风险操作。
- 导入失败、解析失败、无搜索结果、无引用来源、模型未配置等空状态和错误提示清晰。
- 生产环境配置、日志、备份、健康检查、告警和回滚方案已验证。

上线标准：

- 登录、注册或 OIDC 流程可用，异常提示明确。
- 权限边界清晰，无明显越权路径。
- 常见文档类型解析成功，失败时可追踪原因。
- 问答结果能展示来源，或明确说明无法回答。
- 关键异步任务有进度、终态和失败提示。
- 试点业务方完成验收并确认上线范围。

## 维护规则

- 项目资料只记录当前有效决策、当前范围和当前待办。
- 已完成任务进入发布说明或 Git 记录，不保留在执行清单。
- 过期研发计划、专项评审和历史 backlog 不再放入当前资料目录。
- 新需求进入路线图前，必须明确业务价值、目标用户、使用场景和验收标准。
