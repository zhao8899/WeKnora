# WeKnora Roadmap

本文档维护当前有效的产品路线图。已完成事项不再保留为勾选清单；历史实现可通过 Git 记录和发布说明追溯。

## 产品定位

WeKnora 的核心方向是企业知识库底座：把企业内部文档、FAQ、业务资料和问答流程沉淀为可检索、可追溯、可运营的知识服务。

当前阶段优先保证企业内部知识库的一期闭环，而不是扩展成泛 AI 平台。

## 近期优先级

### P0：一期试点稳定化

- 收敛平台超级管理员和普通租户两类角色的首页、菜单、设置和权限边界。
- 优化普通租户主链路：创建知识库、导入内容、FAQ、搜索、问答、共享。
- 优化问答可信度：引用来源、无法回答提示、低质量答案反馈。
- 优化导入体验：进度、失败原因、重试建议、异步任务终态。
- 清理过期项目资料，保持范围、角色和待办实时有效。

### P1：试点运营增强

- 建立试点问题样本集，持续跟踪无结果、低质量答案和知识缺口。
- 增强 FAQ 批量导入、校验、去重和修复反馈。
- 增强租户级用量、内容质量和知识维护提示。
- 完善用户使用指引和上线说明。

### P2：二期候选能力

- 外部数据源连接器和自动同步。
- Agent 编排和工具链。
- GraphRAG 和知识图谱生产级问答。
- MCP 平台化开放。
- 多渠道 IM 接入。
- 运营分析看板和答案质量评估。

## 不作为近期重点

- 复杂研发知识中台。
- 面向普通租户的高级模型参数调试。
- 大规模外部系统 ACL 继承。
- 实验性工具链在前台大面积开放。

## 维护规则

- 路线图只记录未完成和仍有效的方向。
- 已完成任务进入发布说明或 Git 记录，不留在执行清单。
- 新需求进入路线图前必须明确业务价值、目标用户、使用场景和验收标准。
