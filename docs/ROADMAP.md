# WeKnora Roadmap

本文档只维护当前仍有效的后续方向。产品定位、当前范围、角色边界和上线检查统一维护在 [项目资料](./PROJECT.md)。

## P0：试点稳定化

- 收敛平台超级管理员和普通租户两类角色的首页、菜单、设置和权限边界。
- 完成普通租户主链路冒烟：创建知识库、导入内容、FAQ、搜索、问答、引用来源、共享。
- 完成平台超级管理员主链路冒烟：租户管理、平台配置、系统运行状态、关键风险操作。
- 优化问答可信度：引用来源、无法回答提示、低质量答案反馈。
- 优化导入体验：进度、失败原因、重试建议、异步任务终态。
- 确认生产基础项：配置、日志、备份、健康检查、告警、回滚方案。

## P1：试点运营增强

- 建立试点问题样本集，持续追踪无结果、低质量答案和知识缺口。
- 增强 FAQ 批量导入、校验、去重和修复反馈。
- 增强租户级用量、内容质量和知识维护提示。
- 完善用户使用指引和上线说明。

## P2：二期候选能力

- 外部数据源连接器和自动同步。
- Agent 编排和工具链。
- GraphRAG 和知识图谱生产级问答。
- MCP 平台化开放。
- 多渠道 IM 接入。
- 运营分析看板和答案质量评估。

## 维护规则

- 路线图只记录未完成和仍有效的方向。
- 已完成任务进入发布说明或 Git 记录，不保留在执行清单。
- 新需求进入路线图前必须明确业务价值、目标用户、使用场景和验收标准。
