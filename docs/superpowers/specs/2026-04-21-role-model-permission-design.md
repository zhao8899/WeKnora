# 角色权限设计：模型管理（已归档）

**原始日期：** 2026-04-21  
**当前状态：** 已归档  
**归档原因：** 原文使用“租户管理员”表述，容易被误解为独立角色；当前产品口径已统一为两个角色。

## 当前角色口径

当前只保留两个角色：

- 平台超级管理员：通过 `users.can_access_all_tenants = true` 区分，负责平台级配置、租户管理和全局能力边界。
- 普通租户：通过 `users.can_access_all_tenants = false` 区分，租户自己的管理员和使用者；“租户管理员”只是称呼，不是独立角色。

## 模型管理原则

- 平台超级管理员可以管理平台级模型、内置模型和全局配置。
- 普通租户只能管理本租户自有模型或使用平台开放的内置模型。
- 普通租户不能查看平台内置模型的敏感凭证，如 API Key、Base URL 密钥等。
- 普通租户不能修改平台内置模型的全局属性。
