# 知识智能全链路设计（已归档）

**原始日期：** 2026-04-20  
**当前状态：** 已归档  
**归档原因：** 原文是能力扩展型设计，包含 Answer Evidence、健康看板、连接器等多个方向；其中部分能力已落地，部分进入二期候选，不再作为当前一期执行方案。

## 当前结论

WeKnora 当前阶段不以“AI Executive Assistant”作为一期产品表达，而以“企业内部知识库底座”作为一期定位。

一期重点：

- 普通租户能完成知识库建设、内容导入、FAQ 维护、搜索和问答。
- 问答结果具备来源和可信度表达。
- 平台超级管理员能控制租户、配置和系统风险。
- 试点运营能发现知识缺口、低质量答案和维护问题。

## 历史设计中已吸收的方向

- Answer Evidence 和来源反馈。
- 知识健康分析和运营看板。
- 数据源同步治理中的 external_id、配置加密、归档语义。
- 将“堆能力”转向“可信问答、运营治理、用户主链路”的产品判断。
